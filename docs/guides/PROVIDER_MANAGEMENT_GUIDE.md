## 1. التسجيل وإنشاء الحساب

### طريقة التسجيل

1. **الدخول لصفحة التسجيل**: `/provider/join` أو من خلال صفحة الشراكة `/partner`
2. **إدخال البيانات الأساسية**:
   - البريد الإلكتروني
   - كلمة المرور (6 أحرف على الأقل)
   - اختيار المحافظة

### حالات الحساب بعد التسجيل

| الحالة             | الوصف                  | الإجراء المطلوب                 |
| ------------------ | ---------------------- | ------------------------------- |
| `incomplete`       | لم يكتمل ملء البيانات  | إكمال بيانات المتجر             |
| `pending_approval` | بانتظار موافقة الإدارة | الانتظار                        |
| `approved`         | تم القبول              | بدء البيع                       |
| `rejected`         | مرفوض                  | مراجعة سبب الرفض وإعادة التقديم |
| `suspended`        | موقوف مؤقتاً           | التواصل مع الدعم                |

### قبول دعوة الانضمام لفريق العمل

إذا تلقيت دعوة للانضمام لفريق عمل متجر:

1. **افتح رابط الدعوة** المرسل إليك
2. **سجّل حساب جديد** أو **سجّل دخول** إذا كان لديك حساب
3. **راجع الصلاحيات الممنوحة لك**:
   - إدارة الطلبات والاستردادات
   - إدارة المنتجات
   - عرض بيانات العملاء
   - عرض التحليلات
   - إدارة العروض
4. **اقبل الدعوة** لتبدأ العمل

## 2. إكمال بيانات المتجر

### المعلومات المطلوبة

#### معلومات المتجر الأساسية

| الحقل                | الوصف                        | مثال                   |
| -------------------- | ---------------------------- | ---------------------- |
| اسم المتجر (عربي)    | الاسم الذي سيظهر للعملاء     | مطعم سلطان بيتزا       |
| اسم المتجر (إنجليزي) | للعملاء الناطقين بالإنجليزية | Sultan Pizza           |
| رقم الهاتف           | للتواصل (صيغة مصرية)         | 01012345678            |
| العنوان              | عنوان المتجر بالتفصيل        | ش صالح حمام - بني سويف |
| الشعار (Logo)        | صورة الشعار (حد أقصى 2MB)    | صورة PNG/JPG           |

#### الموقع الجغرافي

- **المحافظة**: محددة من التسجيل (لا يمكن تغييرها)
- **المدينة**: اختيار من قائمة مدن المحافظة
- **الموقع على الخريطة**: تحديد الموقع بدقة عبر GPS

#### إعدادات التوصيل

| الإعداد             | الوصف                     | القيمة الافتراضية |
| ------------------- | ------------------------- | ----------------- |
| رسوم التوصيل        | المبلغ المضاف للطلب       | 15 ج.م            |
| وقت التوصيل المتوقع | الوقت التقريبي بالدقائق   | 30-45 دقيقة       |
| الحد الأدنى للطلب   | أقل قيمة مقبولة للطلب     | 50 ج.م            |
| نطاق التوصيل        | المسافة القصوى بالكيلومتر | 5 كم              |

### بعد إكمال البيانات

- يتحول حالة الحساب إلى `pending_approval`
- تراجع الإدارة البيانات والمستندات
- يتم إخطارك بالقرار عبر التطبيق والإيميل

## 4. إدارة المنتجات

### إضافة منتج جديد

**المسار**: `/provider/products/new`

#### البيانات المطلوبة

| الحقل                | مطلوب | الوصف                            |
| -------------------- | ----- | -------------------------------- |
| اسم المنتج (عربي)    | ✅    | الاسم الظاهر للعملاء             |
| اسم المنتج (إنجليزي) | ❌    | للعملاء الناطقين بالإنجليزية     |
| الوصف (عربي)         | ❌    | وصف تفصيلي للمنتج                |
| السعر                | ✅    | السعر الحالي بالجنيه             |
| السعر الأصلي         | ❌    | لإظهار نسبة الخصم                |
| صورة المنتج          | ❌    | صورة مربعة 800×800 بكسل          |
| التصنيف              | ❌    | تصنيف المنتج (بيتزا، مشروبات...) |
| وقت التحضير          | ❌    | الوقت بالدقائق                   |
| متاح                 | ✅    | هل المنتج متاح للطلب؟            |

#### معايير صور المنتجات

```
الأبعاد الموصى بها: 800×800 بكسل (مربع 1:1)
الحد الأدنى: 400×400 بكسل
الحجم الأقصى: 3 ميجابايت
الصيغ المقبولة: JPEG, PNG, WebP
```

**ملاحظة**: يتم قص الصور تلقائياً للحصول على نسبة 1:1 من المركز.

### إدارة المنتجات الحالية

**المسار**: `/provider/products`

#### الإجراءات المتاحة

| الإجراء     | الوصف                     |
| ----------- | ------------------------- |
| تعديل       | تغيير بيانات المنتج       |
| إخفاء/إظهار | إيقاف/تشغيل المنتج مؤقتاً |
| حذف         | إزالة المنتج نهائياً      |

#### الفلاتر والبحث

- **بحث**: بالاسم العربي أو الإنجليزي
- **فلتر التوفر**: الكل / متاح / غير متاح
- **فلتر التصنيف**: حسب تصنيفات المنتجات

### استيراد المنتجات بالجملة

**المسار**: `/provider/menu-import`

1. **حمّل القالب** (Excel أو CSV)
2. **أضف بيانات منتجاتك** في الملف
3. **ارفع الملف** للنظام
4. **راجع البيانات** وصحح الأخطاء
5. **أكد الاستيراد**

## 5. إدارة الطلبات

### مسار الطلب

```
طلب جديد ──► قبول ──► تحضير ──► جاهز ──► قيد التوصيل ──► تم التسليم
    │                                                        │
    └──────────────────── رفض/إلغاء ◄────────────────────────┘
```

### حالات الطلب

| الحالة             | اللون | الوصف        | الإجراء المتاح |
| ------------------ | ----- | ------------ | -------------- |
| `pending`          | أصفر  | طلب جديد     | قبول أو رفض    |
| `accepted`         | أزرق  | تم القبول    | بدء التحضير    |
| `preparing`        | أزرق  | قيد التحضير  | تحديد كجاهز    |
| `ready`            | سماوي | جاهز للتسليم | إرسال للتوصيل  |
| `out_for_delivery` | أزرق  | قيد التوصيل  | تأكيد التسليم  |
| `delivered`        | أخضر  | تم التسليم   | -              |
| `cancelled`        | أحمر  | ملغي         | -              |
| `rejected`         | أحمر  | مرفوض        | -              |

### معلومات الطلب

كل طلب يحتوي على:

- **رقم الطلب**: معرف فريد
- **بيانات العميل**: الاسم، الهاتف
- **عنوان التوصيل**: العنوان الكامل بالتفصيل
- **المنتجات**: قائمة المنتجات مع الكميات والأسعار
- **ملاحظات العميل**: تعليمات خاصة
- **طريقة الدفع**: نقدي أو إلكتروني
- **التوقيت**: وقت الطلب والتوصيل المتوقع

### الطلبات المخصصة (Custom Orders)

**المسار**: `/provider/orders/custom`

إذا فعّلت الطلبات المخصصة، يمكن للعملاء:

- إرسال طلب نصي
- إرسال رسالة صوتية
- إرسال صورة لما يريدونه

**دورك كتاجر**:

1. استلام الطلب المخصص
2. تحديد السعر
3. إرسال عرض السعر للعميل
4. انتظار موافقة العميل
5. تنفيذ الطلب بعد الموافقة

## 6. النظام المالي والعمولات

### فترة السماح (Grace Period)

```
┌────────────────────────────────────────────────────┐
│                   فترة السماح                      │
│                                                    │
│  المدة: أول 3 أشهر من أول طلب                      │
│  العمولة: 0% (صفر)                                 │
│  الهدف: دعم المتجر في بداية انطلاقته              │
│                                                    │
│  ⚠️ غير قابلة للتمديد أو التحويل                   │
└────────────────────────────────────────────────────┘
```

### العمولة القياسية

بعد انتهاء فترة السماح:

| البند        | النسبة                               |
| ------------ | ------------------------------------ |
| العمولة      | 7% كحد أقصى                          |
| تُحسب على    | إجمالي الطلب (بدون رسوم التوصيل)     |
| رسوم التوصيل | حق كامل للمتجر (لا تُخصم منها عمولة) |

#### مثال حسابي

```
إجمالي الطلب:         200 ج.م
رسوم التوصيل:          15 ج.م
─────────────────────────────
المبلغ الخاضع للعمولة: 200 ج.م
العمولة (7%):           14 ج.م
─────────────────────────────
صافي ربح المتجر:       186 ج.م
+ رسوم التوصيل:         15 ج.م
═════════════════════════════
إجمالي للمتجر:         201 ج.م
```

### نموذج الدفع الحالي (الدفع عند الاستلام)

```
┌─────────────────────────────────────────────────────┐
│                  تدفق الأموال                        │
│                                                     │
│  العميل ──[يدفع نقداً]──► المتجر                    │
│                                                     │
│  المتجر ──[يدفع العمولة]──► المنصة (تسوية دورية)    │
└─────────────────────────────────────────────────────┘
```

### تأثير الاسترداد على التسوية

عند حدوث استرداد:

| نوع الاسترداد | تأثير على العمولة         |
| ------------- | ------------------------- |
| استرداد كامل  | إلغاء العمولة بالكامل     |
| استرداد جزئي  | تخفيض العمولة بنفس النسبة |

#### مثال - استرداد جزئي

```
الطلب الأصلي: 200 ج.م | العمولة: 14 ج.م
─────────────────────────────────────
مبلغ الاسترداد: 100 ج.م (50%)
العمولة الجديدة: 7 ج.م (تخفيض 50%)
```

### صفحة المالية

**المسار**: `/provider/finance`

تعرض:

- **ملخص الإيرادات**: إجمالي، عمولات، صافي
- **حالة فترة السماح**: المتبقي أو انتهت
- **تفاصيل التسويات**: السجل الكامل
- **تقسيم طرق الدفع**: نقدي vs إلكتروني

## 7. إعدادات المتجر

### التبويبات المتاحة

**المسار**: `/provider/settings`

#### 1. المتجر (Store)

- تعديل الأسماء والوصف
- تحديث الشعار
- تعديل العنوان

#### 2. ساعات العمل (Working Hours)

| اليوم | من     | إلى     |
| ----- | ------ | ------- |
| السبت | 9:00 ص | 11:00 م |
| الأحد | 9:00 ص | 11:00 م |
| ...   | ...    | ...     |

- تحديد ساعات العمل لكل يوم
- إغلاق أيام معينة
- نسخ الساعات بين الأيام

#### 3. التوصيل (Delivery)

- رسوم التوصيل
- وقت التوصيل المتوقع
- الحد الأدنى للطلب
- نطاق التوصيل

#### 4. الاستلام (Pickup)

- تفعيل/إلغاء خيار الاستلام
- تعليمات الاستلام
- وقت التحضير للاستلام

#### 5. الطلبات المخصصة (Custom Orders)

| الإعداد            | الوصف                        |
| ------------------ | ---------------------------- |
| نوع العمل          | قياسي / مخصص / هجين          |
| قبول طلبات نصية    | نعم/لا                       |
| قبول رسائل صوتية   | نعم/لا                       |
| قبول صور           | نعم/لا                       |
| مهلة التسعير       | عدد الساعات لتقديم عرض السعر |
| مهلة موافقة العميل | عدد الساعات للموافقة         |
| إلغاء تلقائي       | عند انتهاء المهلة            |

#### 6. الحالة (Status)

| الخيار      | الاستخدام              |
| ----------- | ---------------------- |
| مفتوح       | المتجر يعمل بشكل طبيعي |
| مغلق مؤقتاً | إيقاف مؤقت (ساعات)     |
| إجازة       | إغلاق لفترة محددة      |

#### 7. الفريق (Team)

- دعوة أعضاء جدد
- إدارة الصلاحيات
- إزالة أعضاء

#### 8. الحساب (Account)

- تغيير كلمة المرور
- حذف الحساب

## 8. إدارة فريق العمل

### إضافة عضو جديد

1. اذهب إلى **الإعدادات** > **الفريق**
2. اضغط **دعوة عضو**
3. أدخل **البريد الإلكتروني**
4. حدد **الصلاحيات**:
   - إدارة الطلبات والاستردادات
   - إدارة المنتجات
   - عرض بيانات العملاء
   - عرض التحليلات
   - إدارة العروض

5. **أرسل الدعوة**

### إدارة الصلاحيات

| الصلاحية            | الوصف                                |
| ------------------- | ------------------------------------ |
| `manage_orders`     | قبول/رفض الطلبات، معالجة الاستردادات |
| `manage_products`   | إضافة/تعديل/حذف المنتجات             |
| `view_customers`    | رؤية بيانات العملاء                  |
| `view_analytics`    | الوصول للتقارير والإحصائيات          |
| `manage_promotions` | إنشاء وإدارة العروض                  |

## 10. الاستردادات

### أنواع طلبات الاسترداد

**المسار**: `/provider/refunds`

| النوع        | الوصف                    |
| ------------ | ------------------------ |
| منتج ناقص    | لم يصل منتج من الطلب     |
| منتج خاطئ    | وصل منتج مختلف           |
| مشكلة جودة   | المنتج تالف أو غير مطابق |
| لم يصل الطلب | الطلب لم يتم تسليمه      |
| أخرى         | أسباب أخرى               |

### دورة الاسترداد

```
طلب استرداد ──► مراجعة ──► موافقة/رفض ──► رد المبلغ ──► تأكيد العميل
```

### معالجة طلب الاسترداد

1. **استلم إشعار** بطلب الاسترداد
2. **راجع التفاصيل**: السبب، الصور، المنتجات
3. **اتخذ قرار**: موافقة أو رفض
4. **إذا وافقت**: رد المبلغ للعميل (نقدي أو إلكتروني)
5. **انتظر تأكيد** استلام العميل (خلال 48 ساعة)

## 11. التقييمات

### عرض التقييمات

**المسار**: `/provider/reviews`

- **متوسط التقييم**: من 5 نجوم
- **توزيع التقييمات**: عدد كل تقييم (1-5)
- **التقييمات الفردية**: مع تعليقات العملاء

### الرد على التقييمات

يمكنك الرد على تقييمات العملاء:

- اشكر العميل على التقييم الإيجابي
- اعتذر وقدم حلول للتقييم السلبي
- الردود تظهر للعملاء الآخرين

## الأسئلة الشائعة

### كيف أغير سعر منتج؟

اذهب إلى **المنتجات** > اضغط على المنتج > **تعديل** > غيّر السعر > **حفظ**

### كيف أوقف البيع مؤقتاً؟

اذهب إلى **الإعدادات** > **الحالة** > اختر **مغلق مؤقتاً** أو **إجازة**

### كيف أعرف رصيدي من العمولات؟

اذهب إلى **المالية** > تبويب **التسويات** لرؤية الرصيد المستحق

### كيف أضيف موظف جديد؟

اذهب إلى **الإعدادات** > **الفريق** > **دعوة عضو** > أدخل الإيميل والصلاحيات

### ماذا لو تأخرت في تسليم طلب؟

تواصل مع العميل عبر الرسائل في صفحة الطلب واشرح سبب التأخير

### كيف أتعامل مع طلب استرداد؟

اذهب إلى **الاستردادات** > اضغط على الطلب > راجع التفاصيل > قرر الموافقة أو الرفض
